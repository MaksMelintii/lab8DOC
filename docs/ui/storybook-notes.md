# Storybook Notes

## Призначення
Storybook використовується як візуальна документація frontend-компонентів системи бронювання квитків.

## Обрані компоненти

### 1. Button
Використовується для основних дій користувача:
- пошук рейсів;
- створення бронювання;
- оплата;
- скасування.

#### Stories
- Default
- Disabled
- Loading

#### Опис станів
- **Default** — кнопка активна та готова до натискання
- **Disabled** — кнопка неактивна, якщо форма невалідна
- **Loading** — показує, що триває запит до API

---

### 2. BookingCard
Компонент для відображення інформації про бронювання.

#### Stories
- Default
- Paid
- Cancelled
- Error

#### Опис станів
- **Default** — щойно створене бронювання
- **Paid** — бронювання успішно оплачене
- **Cancelled** — бронювання скасоване
- **Error** — не вдалося завантажити дані

## Приклад структури stories
Для кожного UI-компонента повинні бути:
- базовий стан;
- щонайменше один альтернативний стан;
- короткий опис призначення;
- перелік props або основних параметрів.

## Значення Storybook
Storybook допомагає:
- документувати UI незалежно від backend;
- перевіряти стани компонентів;
- спрощувати роботу frontend-розробника та QA;
- використовувати UI-документацію як частину Definition of Done.
