# SSD — System Specification Document

## 1. Призначення системи
REST API сервіс для бронювання квитків призначений для пошуку рейсів, створення бронювань, оплати та перегляду статусу замовлень.

## 2. Основні функції
- FR-01 Реєстрація користувача
- FR-02 Автентифікація
- FR-03 Пошук рейсів
- FR-04 Створення бронювання
- FR-05 Оплата бронювання

## 3. Обмеження та припущення
- система працює через REST API;
- взаємодія відбувається у форматі JSON;
- оплата залежить від зовнішнього платіжного сервісу.

## 4. Нефункціональні вимоги
### Продуктивність
- до 300 мс на типовий запит
- не менше 500 запитів на хвилину

### Доступність
- 99.9%

### Безпека
- JWT-аутентифікація
- HTTPS
